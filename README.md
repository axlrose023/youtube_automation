# YouTube Emulation — User Behaviour Flow

## Обзор

Эмулятор имитирует реального пользователя YouTube с двумя режимами поведения,
системой усталости, циклами фокуса, «личностью» сессии и защитой от зацикливания.

---

## Два режима

### Mode A — «Развлечься / фон»

Пользователь не знает, что хочет смотреть. Листает ленту, кликает по
рекомендациям, залипает на длинные видео. Рекламу терпит.

### Mode B — «Решить задачу»

У пользователя есть цель. Ищет конкретные темы, быстро оценивает видео,
ускоряет воспроизведение, перематывает воду, пропускает рекламу сразу.

Режим выбирается случайно при старте и переключается как между циклами,
так и внутри цикла (каждые ~5 действий) в зависимости от усталости.

---

## Личность сессии (SessionPersonality)

При старте каждой сессии генерируется набор параметров «личности» —
Gaussian(mean, std) с клемпом. Две сессии ведут себя заметно по-разному.

| Параметр       | Диапазон  | На что влияет                                      |
|----------------|-----------|-----------------------------------------------------|
| pace           | 0.6–1.4   | Множитель ко всем задержкам (быстрый/медленный юзер)|
| patience       | 0.5–1.5   | Множитель к длительности просмотра видео            |
| focus_span     | 0.7–1.3   | Множитель к длине цикла (короткие/длинные сессии)   |
| search_style   | 0.0–1.0   | 0 = последовательный выбор тем, 1 = случайный       |
| ad_tolerance   | 0.3–1.0   | Множитель к времени ожидания рекламы                |

## Жизненный цикл сессии

```
Старт
  │
  ▼
Открыть YouTube → проверить наличие контента в ленте
  │
  ├─ Есть видео → сканировать превью (3-8с)
  └─ Пустой профиль → сразу к поиску
  │
  ▼
┌─────── ЦИКЛ (вариативная длина × focus_span) ────────────┐
│                                                           │
│  30% шанс: короткий burst (5-15 мин)                     │
│  70% шанс: глубокий dive  (20-50 мин)                    │
│  × personality.focus_span                                 │
│       │                                                   │
│       ▼                                                   │
│  Выбрать действие по весам текущего режима                 │
│       │                                                   │
│       ▼                                                   │
│  Выполнить действие                                       │
│       │                                                   │
│       ▼                                                   │
│  Обновить fatigue (нелинейный), синхронизировать Redis     │
│       │                                                   │
│       ▼                                                   │
│  Каждые ~5 действий: проверка смены режима (A↔B)          │
│       │                                                   │
│       ▼                                                   │
│  Пауза 0.5-2с × pace между действиями                    │
│       │                                                   │
│       ▼                                                   │
│  Цикл не кончился? → повторить                            │
│                                                           │
└───────────────────────────────────────────────────────────┘
  │
  ▼
Перерыв (5-10% от сессии, макс 3-7 мин) — 20% шанс пропустить
  │
  ▼
Смена режима (A↔B) по уровню fatigue
  │
  ▼
Следующий цикл... (пока не истечёт duration_minutes)
  │
  ▼
Финализация: подсчёт трафика через CDP, возврат результатов
```

## Действия Mode A (развлечение)

| Действие           | Вес  | Описание                                        |
|--------------------|------|-------------------------------------------------|
| watch_long         | 30   | Смотреть видео долго (30с-10мин, по duration)   |
| click_recommended  | 30   | Кликнуть рекомендованное видео                  |
| scroll_feed        | 15   | Прокрутить ленту 2-5 раз                        |
| search             | 10   | Поискать одну из тем (0 если все уже искались)  |
| idle               | 5    | Пауза 2-8с, пошевелить мышкой                  |
| go_home            | 3    | Вернуться на главную                            |

При fatigue > 0.6: idle и watch_long растут, search отключается.

## Действия Mode B (задача)

| Действие           | Вес  | Описание                                        |
|--------------------|------|-------------------------------------------------|
| watch_focused      | 30   | Активный просмотр: оценка → ускорение → досмотр |
| search             | 20   | Поиск следующей темы (3 если все искались)      |
| surf_video         | 18   | Быстрая проверка (10-40с), уйти если не то      |
| click_recommended  | 12   | Кликнуть рекомендованное (если релевантно)      |
| scroll_results     | 8    | Прокрутить результаты поиска                    |
| idle               | 3    | Короткая пауза                                 |
| go_back            | 3    | Назад к результатам                             |

При fatigue > 0.5: больше surf_video и idle (меньше терпения).

## Принудительные переходы

После поиска **всегда** идёт клик на видео:
- Mode A → `watch_long`
- Mode B → `watch_focused`

При первом запуске (нет searched_topics) — принудительный `search`.

Покрытие тем теперь **динамическое**:
- если времени мало или тем осталось много, растёт шанс `search` (`dynamic coverage push`);
- если времени достаточно, допускаются дополнительные действия без жёсткого "каждая тема = ровно 25%".

Re-anchor к теме:
- при `recommended_streak` лимите → `search`;
- периодически после нескольких действий без поиска → `search`;
- если был exploratory-клик в рекомендации/shorts (`topic_drifted`) → следующий шаг уводит обратно в `search`.

При surf_streak >= 4 → `refine_search` (если есть unsearched) или `click_recommended`.

### refine_search

В отличие от `search` (новая тема), `refine_search` модифицирует последний запрос
добавляя модификатор: `"cats" → "cats tutorial"`, `"cats how to"`, `"cats best"` и т.д.

Модификаторы: tutorial, how to, best, review, explained, for beginners, tips, guide, vs, top 10.

## Просмотр видео — как решается длительность

```
Открыл видео
  │
  ▼
Обработка рекламы (patient / impatient)
  │
  ▼
Запуск воспроизведения (JS video.play() если paused)
  │
  ▼
Получить video.duration из DOM
  │
  ▼
Шаг 1: Решить — досматривать или нет (см. таблицу вероятностей ниже)
  │
  ├─ Решил досмотреть → смотрит 85-100% от video.duration
  │
  └─ Решил НЕ досматривать → смотрит часть:
       ├─ Видео <3 мин:   40-80% длины
       ├─ Видео 3-10 мин: 20-60% длины
       └─ Видео >10 мин:  10-40% длины
  │
  ▼
Шаг 2: Умножить на personality.patience (0.5–1.5)
  │
  ▼
Шаг 3: Применить модификатор fatigue (уменьшить время если устал)
  │
  ▼
Шаг 4: Если не все темы покрыты — применить динамический cap
       (зависит от remaining time и remaining topics)
  │
  ▼
Шаг 5: Ограничить оставшимся временем сессии
  │
  ▼
Смотреть с микро-взаимодействиями
```

### Вероятность решения «досмотреть до конца»

Шанс что бот **решит** досмотреть видео полностью (шаг 1):

|                    | < 2 мин | 2-10 мин | > 10 мин |
|--------------------|---------|----------|----------|
| **Mode A**         | 35%     | 20%      | 10%      |
| **Mode B**         | 20%     | 10%      | 5%       |

Если решил — смотрит 85-100% видео. Если нет — смотрит частичную долю (см. выше).

Модификаторы вероятности:
- Fatigue снижает шанс (множитель `1 - fatigue × 0.5`, минимум ×0.3)
- Видео > 20 мин при fatigue > 0.5 → никогда не досматривает

---

## Усталость (fatigue)

Значение от 0.0 до 1.0. Растёт **нелинейно** по формуле `(elapsed/total)^0.8 + шум(±2%)`.
Быстрее нарастает в середине сессии, замедляется к концу.

| Fatigue | Эффект                                              |
|---------|-----------------------------------------------------|
| 0.0-0.3 | Нормальное поведение                               |
| 0.3-0.5 | Mode A может переключиться на Mode B (отдохнул)    |
| 0.5-0.6 | Mode B: больше surf, меньше терпения               |
| 0.6-0.8 | Mode B → A (устал). Больше idle и пассивности      |
| 0.8-1.0 | Сильный сдвиг к пассивности: watch_long +20,       |
|         | idle +15, scroll +10, search отключён              |

## Работа с рекламой

| Ситуация                     | Mode A                              | Mode B                |
|------------------------------|-------------------------------------|-----------------------|
| Pre-roll (пропускаемый)      | Ждёт 3-12с × ad_tolerance, потом пропускает | Пропускает сразу × ad_tolerance |
| Pre-roll (непропускаемый)    | Смотрит                            | Ждёт, пытается снова  |
| Mid-roll                     | 40% смотрит 3-8с, потом пропускает | Пропускает сразу      |

Все задержки при рекламе масштабируются `personality.ad_tolerance` (0.3–1.0).

## Mode B — активный просмотр (watch_focused)

```
Открыл видео
  │
  ▼
Пропустить рекламу
  │
  ▼
Запустить воспроизведение
  │
  ▼
Первичная оценка (15-45с)
  │
  ├─ 25%: не зашло → уходит (surf_streak +1, только если > 90с осталось)
  │
  ▼ 75%: остаётся
  │
  ├─ 40%: ускоряет (1.25x-2x)
  ├─ 30%: перематывает вступление (+10-30с)
  │
  ▼
Основной просмотр (может досмотреть до конца)
  │
  ▼
Микро-взаимодействия во время просмотра:
  ├─ 10%: скролл комментариев
  ├─ 5%: пошевелить мышкой
  └─ 3%: пауза (2-6с) — «записать/повторить»
  │
  ▼
Сброс скорости на 1.0x
```

## Защита от зацикливания

| Ситуация                      | Лимит | Реакция                                         |
|-------------------------------|-------|--------------------------------------------------|
| Ошибки подряд                 | 4     | Принудительный переход на главную                |
| Surf-выходы подряд (Mode B)   | 4     | Смена запроса или рекомендация                   |
| Не найдены видео (на /watch)  | 1     | recover_from_no_video + safe_go_home             |
| Не найдены видео (повторно)   | 2     | safe_go_home + в ActionPicker: search или go_home|
| Таймаут go_back               | —     | Fallback на safe_go_home                         |
| Навигация на about:blank      | —     | Автоматический safe_go_home                      |
| Клик промахнулся              | —     | Fallback: навигация по href элемента             |

### Поиск видео на странице

1. Если scrollY > 900 — скролл вверх к началу (Home + scrollTo(0,0))
2. На /watch странице — скролл к #secondary (рекомендации)
3. Проверка селекторов: query → wait_for_selector (700ms) → query
4. Фильтрация по href (`/watch` или `/shorts/`) + режим `allow_shorts`
   (в topical-режиме шорты обычно выключены, в exploratory — могут включаться)
5. Проверка bounding_box > 0 (видимость)
6. Если ничего — повторная попытка с force reset
7. Выбор случайного элемента из найденных

## API

### Запуск

```
POST /emulation/start
{
  "duration_minutes": 60,
  "topics": ["книги", "детективы", "Nike", "спорт"]
}

→ { "session_id": "uuid", "status": "queued" }
```

### Статус

```
GET /emulation/{session_id}/status

→ {
    "session_id": "uuid",
    "status": "running",
    "elapsed_minutes": 12.3,
    "bytes_downloaded": 154890234,
    "topics_searched": ["книги", "детективы"],
    "videos_watched": 5,
    "mode": "entertainment",
    "fatigue": 0.21,
    "error": null
  }
```
